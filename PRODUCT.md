# Product

## Register

product

## Users

Single user (personal project). The owner knows the interface and the data. Context is desktop, seated, reviewing finances deliberately. The job to be done: scan current financial state, spot anomalies, log a transaction, understand a trend. Speed and clarity beat discoverability. No onboarding ceremony needed.

## Product Purpose

MoneyMapper is a personal finance dashboard that tracks income, expenses, budgets, and spending patterns over time. Success looks like opening the app and immediately knowing where you stand financially, without hunting or interpreting.

## Brand Personality

Precise, minimal, honest. The interface is a tool, not an experience. It respects the user's intelligence and doesn't editorialize about their money. Nothing soft, nothing celebratory, nothing gamified.

## Anti-references

- **Mint / YNAB**: pastel palettes, rounded everything, hand-holding copy, softened financial reality with consumer-friendly aesthetics.
- **Robinhood**: inflated numbers, green gradient hype, gamified investment-app energy that turns portfolio value into a scoreboard.
- **Generic SaaS dashboard**: cream or slate-100 background, white cards with gray borders and indigo accent, shadow-sm on everything. The exact AI-generated template aesthetic this project wants to leave behind.
- **Banking apps**: corporate sterility with zero craft. Cold, institutional, no considered typography or layout.

## Design Principles

1. **Data before decoration.** Every visual element must carry information or aid scanning. Decoration that doesn't convey meaning is removed, not softened.
2. **Confident density.** Show the user what they need without burying it. Information density is a feature. Hiding data behind extra clicks or collapsibles is a last resort, not a default.
3. **Numbers are the product.** Typography and layout exist to make financial figures scannable at a glance. The number is the hero; everything else supports it.
4. **Consistent vocabulary.** One button shape, one form control style, one icon family, one way to show a trend. The interface should feel like a single designed system, not assembled from parts.
5. **Quiet intelligence.** Motion and color serve state changes and data meaning, not aesthetics. When something moves or changes color, it communicates something specific.

## Accessibility & Inclusion

WCAG AA minimum. Body text ≥ 4.5:1 contrast against background. Interactive elements have visible focus states. Chart data is not conveyed by color alone. Respects `prefers-reduced-motion`.
